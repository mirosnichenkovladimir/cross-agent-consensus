# Glossary

## Agent roles

- Orchestrator: Coordinates rounds, preserves state, enforces policy, and decides when to continue, stop, or escalate.
- Author Agent: Produces or revises an artifact and responds to findings.
- Reviewer Agent: Reviews an artifact version and emits findings. Reviewers do not apply changes in the base protocol.
- Validator: Produces external evidence such as tests, lint, schema checks, screenshots, or structural checks.
- Human Supervisor: Resolves escalations and approves final outcomes when required.

## Artifacts

- Artifact: The thing being produced or reviewed: document, design, code diff, UI, plan, etc.
- Artifact Version: Immutable reference to one version of an artifact.
- Task Brief: Original task context and success criteria.
- Policy: Per-run rules for severity, materiality, round limits, validators, and escalation.

## Finding objects

- Raw Finding: Immutable reviewer output tied to one artifact version.
- Canonical Finding: Normalized lifecycle object that may aggregate related raw findings.
- Author Response: Author's explicit response to a canonical finding.
- Re-Review Decision: Reviewer decision after inspecting author response and any revised artifact.
- Validation Evidence: Non-opinion evidence generated by validators.

## Outcomes

- Consensus: No unresolved material findings remain and required validators pass.
- Escalation: Human decision is needed.
- Abort: The run cannot continue safely or meaningfully.
- Round-Limit Stop: Max rounds reached before consensus.
